### ✅ 6. 修复图片悬浮工具栏失效问题
**文件**: `apps/web/src/app/canvas/StarCanvas.tsx`

**问题**: `StarCanvas` 自己的 `imageHoverRegistry` 和 `ImageNode.tsx` 内部的 `imageHoverRegistry` 是两个不同对象，导致悬浮事件无法触发。
**修复**: 删除 StarCanvas 内的本地定义，改为从 `ImageNode.tsx` 导入 `registerImageHoverHandlers` 和 `unregisterImageHoverHandlers`，在 `useEffect` 中动态注册/注销。

```tsx
// 新增 import
import ImageNode, { registerImageHoverHandlers, unregisterImageHoverHandlers } from "./components/nodes/ImageNode"

// useEffect - 监听 nodes 变化
useEffect(() => {
  const imageNodes = nodes.filter((node) => node.type === "image")
  imageNodes.forEach((node) => {
    registerImageHoverHandlers(node.id, {
      onMouseEnter: handleImageNodeMouseEnter,
      onMouseLeave: handleImageNodeMouseLeave,
    })
  })
  return () => {
    imageNodes.forEach((node) => unregisterImageHoverHandlers(node.id))
  }
}, [nodes, handleImageNodeMouseEnter, handleImageNodeMouseLeave])
```

- `curl` 得到的 HTML 包含 `BAILOUT_TO_CLIENT_SIDE_RENDERING`（SSR 回退到 CSR），提示 `next/dynamic` 导致

### ⚠️ 2. `allowedDevOrigins` 跨域 HMR 警告
**文件**: `apps/web/next.config.ts`

Next.js 阻止了来自 `127.0.0.1` 的 HMR WebSocket 连接（允许的是 `localhost`）。不影响功能，但会在 dev log 里一直刷警告。

**修复方法**（在 `next.config.ts` 加一行）:
```ts
const nextConfig: NextConfig = {
  outputFileTracingRoot: path.join(__dirname, "../.."),
  allowedDevOrigins: ["127.0.0.1"],  // ← 加这行
}
```
加了之后需要重启 dev server。
