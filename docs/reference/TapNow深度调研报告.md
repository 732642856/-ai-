# TapNow 深度调研报告

## 一、项目概述

TapNow 是一个面向创作者和企业的新一代 **AI 视觉创作引擎**，核心形态为**节点式无限画布工作流工具**。它有两个主要版本：

| 维度 | 开源版 Tapnow (LinXueyuanStdio) | 商业版 TapNow Studio |
|:---|:---|:---|
| **定位** | 偏创作工作流的可视化前端工具 | 面向企业和创作者的商业AI视觉创作引擎 |
| **架构** | 单 HTML 文件，无需构建 | 云端 SaaS 平台 |
| **协议** | GPL v3 | 商业付费（按量/订阅） |
| **模型支持** | Midjourney、即梦、Sora、Grok、GPT-4o-image、Gemini | 15+ 模型：Veo3、Sora2 Pro、可灵、即梦4.0、Midjourney、Flux 等 |
| **社区** | 无内置社区 | TapTV 社区（12万+ 公开工作流） |
| **一致性控制** | 基础参考图管理 | Banana 模型驱动高保真一致性 |
| **Cinema Lab** | 无 | 有（多角度控制、火柴人、焦点编辑、摄影机控制） |

## 二、TapNow 完整功能清单

### 2.1 开源版 Tapnow 功能清单

#### ♾️ 无限画布与节点系统
- 拖拽式连线，串联图片、视频、文本和生成节点
- 无限缩放与平移，支持大画布布局
- 多选、框选、批量移动、删除和自动整理节点
- 撤销/重做、运行状态提示、错误反馈
- 节点级预览（单击选图、双击大图灯箱、左右/上下切换）

#### 🎞️ 视频理解与智能分镜
- 视频输入 / 外部视频 URL 加载
- 自动抽帧、关键帧整理与选择
- 镜头组织、视频拆解、提示词反推
- 智能分镜表、镜头卡片、表格视图
- 口播提取、镜头描述、镜头级提示词
- 参考图管理

#### 🎨 AI 绘图工作流
- **Midjourney 深度集成**：参考图、角色一致性、风格一致性链路
- **即梦图像模型**：多代模型、图生图
- **通用图像模型**：gpt-4o-image、gemini 图像入口
- 多图输入、小图预览、主图切换、批次结果聚合
- 图像对比节点与预览窗口联动

#### 🎥 AI 视频生成
- 文生视频、图生视频、首尾帧视频生成
- 模型级时长、比例、分辨率配置
- 异步轮询配置
- 内置 Sora、Jimeng Video、Grok Video 等模型入口

#### 📚 小说 / 脚本 / 角色 / 场景
- 小说输入节点（长文本脚本源）
- 角色提取节点、场景提取节点
- 角色库与场景信息管理
- LLM Prompt 记忆槽
- 脚本拆分与回填

#### 🛠️ 资产与本地协同
- 历史记录、批量下载/删除、发送到画布
- 本地缓存与缓存刷新
- 保存到本地节点
- 项目保存、工作流导入导出、资产打包保存
- 中英切换、主题切换

### 2.2 商业版 TapNow Studio 独有功能

#### ① Tapflow 画布（节点编排）
- 粗粒度功能块节点：文本 → 图像 → 视频 → 音频
- 拖拽式连接，像拼乐高一样搭建工作流
- `/` 快捷召唤：三视图、画面推演、九宫格、电影级光影

#### ② 多模型中枢
- 15+ 顶尖模型任意切换
- 深度集成国产模型：可灵、Seedance、即梦、Hailuo
- 跨模型调度，取长补短

#### ③ 一键拉片
- 上传竞品 TVC/参考视频
- AI 反向拆解：秒级生成可编辑分镜表 + 提示词
- 视频解析后一键克隆完整工作流

#### ④ Cinema Lab（专业影视控制）
| 功能 | 说明 | 操作方式 |
|:---|:---|:---|
| **多角度控制** | 精确控制角色朝向和姿态 | 竖直/水平拖拽方块调整角度，无需提示词 |
| **火柴人动作参考** | 自定义角色动作 | 手动捏制火柴人姿势作为参考图 |
| **焦点编辑（魔法棒）** | 精修画面局部细节 | 选择元素精准编辑，如添加蝴蝶结 |
| **摄影机控制** | 模拟电影感镜头效果 | 直接添加景深和特定镜头语言 |
| **高清放大** | 提升视频分辨率 | 生成视频后直接选择高清放大 |

#### ⑤ 高保真一致性（Banana 模型 / NBP 工作流）
- 商品/角色跨镜头"零变脸"
- 多角色、多场景、多角度一致性保持
- 满足品牌安全诉求

#### ⑥ 电商极速套图
- 2 分钟出 20 张主图 + 详情页
- 3 张产品图 + 2 张场景图 → 自动布景、换模特、换风格

#### ⑦ 音画同步
- 内置音频节点
- AI 音乐生成
- 文字转语音（多种音色可选）
- 48kHz 商用版权曲库

#### ⑧ TapTV 社区生态
- 12万+ 公开工作流
- 可查看每一步提示词、模型选择、参数设置
- 一键克隆他人画布
- 发布作品获积分奖励

## 三、用户操作流程（Step by Step）

### 3.1 开源版 Tapnow 操作流程

```
【阶段一：环境准备】
1. 下载 index.html，用浏览器直接打开（或访问在线版）
2. 如需使用即梦：部署本地代理服务 → 获取 sessionid → 配置 API 设置

【阶段二：画布基础操作】
3. 双击画布空白处或右键 → 添加节点（图片输入 / 小说输入 / AI 绘图 / 预览）
4. 从上游节点右侧连接点 → 拖线 → 下游节点左侧输入点，建立数据流
5. 滚轮缩放、按住空白拖拽平移、拖动节点调整布局、框选批量移动

【阶段三：内容创作流程】
6. 小说/脚本输入 → 脚本拆分节点 → 角色提取 → 场景提取
7. 图片输入 / AI 绘图节点（Midjourney/即梦/GPT-4o-image）→ 填写提示词 → 点击执行
8. 图像对比节点 → 预览窗口联动 → 选择满意结果
9. 视频输入 → 自动抽帧 → 关键帧选择 → AI 视频生成（文生视频/图生视频/首尾帧）
10. 历史记录区 → 批量下载 / 发送到画布 / 保存到本地

【阶段四：项目保存】
11. 项目保存（完整状态快照）/ 工作流导出（节点与连接的复用模板）
```

### 3.2 商业版 TapNow Studio 完整工作流

#### 案例一：复刻视频（学习模式）

```
【步骤一：灵感获取】
1. 在 TapTV 社区浏览目标视频
2. 点击【查看创作过程】→【克隆项目】
3. 获取完整画布、节点设置、参数和提示词

【步骤二：视频拆解】
4. 上传参考视频 → 点击【解析】
5. AI 一键拆解分镜 → 查看创作思路

【步骤三：角色设计】
6. 生成角色设计图 → 选择基准图（中景/面部清晰）
7. 使用 / 召唤功能 → 生成三视图
8. 全部角色人设图保存为参考

【步骤四：分镜规划】
9. 输出故事梗概 → 生成九宫格分镜
10. 用人物中景 + 全部角色人设图作为参考，保持角色一致性

【步骤五：图片精修】
11. 九宫格延伸 → 多角度调整角色朝向
12. 火柴人动作参考 → 焦点编辑（魔法棒）精修细节
13. 排好分镜顺序 → 添加镜头语言

【步骤六：视频生成】
14. 图生视频 → 首尾帧控制位置移动
15. 连续镜头体现流畅感（注意动态元素连贯性）

【步骤七：后期合成】
16. 生成背景音乐（AI 参考九宫格写提示词）
17. 文字转语音（高级设置选音色）
18. 高清放大 → 导出成片
```

#### 案例二：原创广告（电商场景）

```
【高效技巧】
1. 批量生成提示词：先一次性生成所有角色的三视图提示词，再依次生成图片（省 2 倍积分）
2. 九宫格放大后仔细核对角色细节（辫子、领子、纹理等）
3. 非必要场景优先 2K，避免 4K 大小限制
4. 元素逐次出现：先去掉食材，再让 AI 逐一生成
```

## 五、TapNow 值得我们学习的具体设计点

### 5.1 产品架构层

| 设计点 | 说明 | 对 StarCanvas 的启示 |
|:---|:---|:---|
| **单文件分发策略** | 开源版仅一个 index.html，零部署成本 | 可考虑提供单文件演示版，降低用户体验门槛 |
| **画布即工作流** | 所有创作在一个无限画布完成，无需跳转 | 进一步强化"一张画布包所有"的理念 |
| **粗粒度节点** | 节点是功能块而非底层参数，降低使用门槛 | 在保持专业性的同时，提供"简化模式"或"专家模式"切换 |
| **一键拉片** | 上传视频 → AI 拆解分镜 + 提示词，将学习成本降到最低 | 可增加"参考视频解析"节点，自动提取分镜和角色 |

### 5.2 创作流程层

| 设计点 | 说明 | 对 StarCanvas 的启示 |
|:---|:---|:---|
| **一键克隆工作流** | 社区作品可完整复制画布、参数、提示词 | 建立作品/工作流市场，支持一键复用 |
| **视频解析拆解** | 上传视频后一键拆解分镜，查看创作思路 | 增加"逆向工程"能力，从成品反推创作流程 |
| **NBP/Banana 一致性** | 跨镜头角色一致性控制，解决 AI 视频最大痛点 | 将 ContinuityGuard 与视频生成深度整合，确保角色不"变脸" |
| **首尾帧控制** | 用两张图控制视频起止，实现位置移动和场景过渡 | 在视频生成节点中增加首尾帧输入 |
| **九宫格推演** | `/` 快捷召唤三视图、画面推演、电影级光影 | 增加快捷命令系统，快速调用常用模板 |

### 5.3 Cinema Lab 专业控制（最值得学习）

| 功能 | 价值 | 实现思路 |
|:---|:---|:---|
| **多角度控制** | 无需提示词，拖拽方块即可调整角色朝向 | 集成 ControlNet / Pose 控制到节点中，可视化调节 |
| **火柴人动作参考** | 用户捏制姿势作为参考图，避免"抽卡" | 增加姿态编辑节点，支持简单骨骼编辑 |
| **焦点编辑（魔法棒）** | 精准选择画面元素进行局部修改 | 集成 Inpainting / 区域编辑到预览节点 |
| **摄影机控制** | 直接添加景深和镜头语言，获得电影感 | 视频生成节点增加镜头运动参数（推拉摇移、景深） |
| **高清放大** | 视频生成后分辨率不足时的补救措施 | 增加视频超分/放大节点 |

### 5.4 社区与生态层

| 设计点 | 说明 | 对 StarCanvas 的启示 |
|:---|:---|:---|
| **TapTV 社区** | 12万+公开工作流，每一步都可见可学 | 建立创作者社区，鼓励分享完整工作流和提示词 |
| **积分激励** | 发布作品、邀请好友获得积分奖励 | 设计创作者激励体系，促进内容生产 |
| **作品即教程** | 看别人的完整工作流比看文字教程更直接 | 支持工作流附带创作说明，形成"可交互教程" |
| **一键克隆 + 再创作** | 克隆后直接在原工作流基础上修改 | 支持工作流 Fork 机制，记录创作 lineage |

### 5.5 交互细节层

| 设计点 | 说明 | 对 StarCanvas 的启示 |
|:---|:---|:---|
| **预览交互** | 单击选图、双击灯箱、左右切换同组、上下切换分镜/历史 | 优化预览节点的交互体验，支持更丰富的浏览方式 |
| **批量操作** | 框选多个节点进行批量移动/删除 | 增加框选和批量操作能力 |
| **本地缓存** | 浏览器本地存储，支持缓存刷新 | 增加本地自动保存和版本历史 |

## 七、关键结论

1. **TapNow 的核心差异化**在于将"专业影视流程工具化"——它不是单一的 AI 生成工具，而是把导演思维产品化的"AI 制片厂"。

2. **最值得 StarCanvas 学习的三点**：
   - **Cinema Lab 的专业控制层**：多角度、火柴人、焦点编辑、摄影机控制，将 AI 生成从"抽卡"变为"可控创作"
   - **一键拉片 + 工作流克隆**：大幅降低学习成本，让复刻成为最高效的学习方式
   - **TapTV 社区生态**：作品即模板、工作流即教程，形成自增长的创作者网络

3. **StarCanvas 的相对优势**：紫微斗数角色设计、ContinuityGuard 连续性检查、角色 Bible 系统——这些是 TapNow 没有的"玄学+编剧"深度融合能力，应继续强化。

4. **融合建议**：StarCanvas 可以在保持命理+编剧特色的基础上，吸收 TapNow 的 Cinema Lab 控制理念、社区克隆机制和一键拉片能力，形成"有灵魂的角色 + 有控制的生成"的独特定位。
