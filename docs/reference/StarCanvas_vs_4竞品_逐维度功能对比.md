## 一、创作层：输入方式 / 模板系统 / Prompt方式

### 小云雀短剧Agent 2.0

| 功能点 | 交互细节 | SC对比 |
|--------|----------|--------|
| **1.1 剧本上传** | 支持上传10万字docx剧本文件，自动解析世界观、时间线、人物关系 | ✅ `ScriptImportPanel` + `analyze-script` API |
| **1.2 大纲AI生成剧本** | 输入故事背景/核心设定→AI自动生成完整剧本，最多一次性生成5集 | ❌ 无此功能，仅支持已有剧本导入 |
| **1.3 100+影视级画风预设** | 内置柯达暖黄胶片、90年代写实电影、韩国冷淡风等风格，可自定义输入 | ⚠️ `VisualStyleBiblePanel` 存在但需手动创建，无预置风格库 |
| **1.4 自定义集数/时长/画幅比** | 上传时一次性配置：时长、画风、横竖屏比例、集数 | ⚠️ 画幅比在节点中有比例选择，但无项目级全局配置 |
| **1.5 旁白一键改编** | 演绎类剧本→解说类剧本自动转换 | ❌ 无此功能 |
| **1.6 系列短剧连续生成** | 一次解析剧本→多剧集连续生成，不需重复上传素材 | ❌ 无系列剧管理，每次为独立项目 |

### TapNow

| 功能点 | 交互细节 | SC对比 |

| **1.8 一键克隆他人工作流** | TapTV社区点击「克隆项目」→完整工作流复制到个人画布 | ❌ 无社区，无可克隆机制 |
| **1.9 视频上传AI解析（一键拉片）** | 上传视频→点击「创建分镜」→2-3分钟输出分镜/运镜/景别/内容表格 | ❌ `video-analysis` API为mock，无此能力 |
| **1.10 参考图搜索（对话内100张）** | 右侧对话框搜索参考图→浏览100张→点击大图「应用到画布」 | ❌ 无集成参考图搜索 |

### ArcReel

| **1.15 渐进式分集规划** | peek探测→Agent建议断点→用户确认→物理切分 | ❌ 无分集规划 |

| **1.17 风格参考图** | 上传风格图→AI自动分析并统一应用到所有图片生成 | ⚠️ VisualStyleBible存在但无自动风格提取 |

### Moyin Creator

## 总结：StarCanvas的定位缺口

StarCanvas目前的强项是**Bible体系**（角色/场景/视觉风格）和**节点式画布基础**，但存在三大结构性缺失：

1. **缺少"产出闭环"**：无法导出成品视频，无法导入剪映，只能停留在"生成素材"阶段
2. **缺少"专业控制"**：没有独立的光影/镜头/摄影机参数面板，生成结果靠碰运气
3. **缺少"智能编排"**：Workflow执行是顺序脚本，不是真正的Multi-Agent自感知编排

这三点恰好是小云雀2.0（控制）、ArcReel（编排）、TapNow（闭环）各自最擅长的方向。
