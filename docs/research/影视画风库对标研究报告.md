## 一、推荐的风格分类体系

基于本次对4个对标应用和多个开源项目的研究，建议采用以下**三层分类体系**：

```
[一级] 画风大类（介质层面）
├── 真人写实 (Live-Action Realism)
├── 动画/漫画 (Animation/Manga)
├── 实验/艺术 (Experimental/Artistic)
└── 特效/幻想 (VFX/Fantasy)

[二级] 电影感子类（审美传统）
真人写实/
├── 港风/王家卫 (90s Hong Kong Art Film)
├── 好莱坞史诗 (Hollywood Epic / IMAX 70mm)
├── 韩国冷淡风 (Korean Cold Tone)
├── 法国新浪潮 (French New Wave)
├── 日式胶片感 (Japanese Film Grain)
├── 美式复古 (American Retro)
├── 纪录片/伪纪录 (Documentary/Mockumentary)
├── 竖屏爽剧 (Vertical Mini-Drama)
└── 广告质感 (Commercial/Aesthetic)

动画/漫画/
├── 2D手绘动画 (2D Traditional)
├── 3DCG建模 (3D CGI)
├── 日式动漫 (Anime)
├── 定格动画 (Stop-Motion / Clay)
└── 水墨动画 (Ink-Wash)

实验/艺术/
├── 赛博朋克 (Cyberpunk)
├── 后印象派/梵高 (Post-Impressionist/Van Gogh)
├── 超现实主义 (Surrealist)
└── 蒸汽波/Y2K (Vaporwave/Y2K)

[三级] 具体子风格（可搜索标签）
示例：王家卫 → 黄绿色调 + 高ISO颗粒 + 抽帧效果 + 浅景深
     好莱坞史诗 → Desaturated + Gritty + Epic Scale + Hans Zimmer tension
     竖屏爽剧 → 高饱和滤镜 + 极致面部特写 + 竖屏9:16
```

### 分类体系设计原则

1. **一级（介质）**：用户最直观的认知。真人/动画/实验/特效 → 决定底层模型选型
2. **二级（审美流派）**：对标市场认知。港风/好莱坞/赛博朋克等 → 用户可以立刻想象
3. **三级（子风格标签）**：技术可执行。具体的光色参数/prompt关键词 → 机器可理解

## 二、各对标应用的风格实现方式

### 2.1 小云雀短剧 Agent 2.0

| 维度 | 详情 |
|------|------|
| **规模** | 100+ 影视级画风 |
| **组织方式** | 大类（如"真人"）+ 细分风格 |
| **已确认风格** | 90年代中国农村、美式复古、韩国冷淡风、柯达暖黄胶片复古、90年代写实电影、日本黑白胶片摄影 |
| **实现方式** | **自研生图模型 + Prompt驱动**。非简单滤镜/LUT，切换风格后场景情绪完全不同 |
| **额外能力** | 支持自定义风格输入、720°全景图生成辅助一致性、光影参数面板（方位/亮度/质感/色温） |
| **技术推断** | 可能采用Fine-tuned生图模型（Seedance 2.0底层），不同风格对应不同的模型权重或LoRA |

### 2.2 TapNow / Tapnow Studio

| 维度 | 详情 |
|------|------|
| **风格系统** | **通过模型切换实现风格差异**。集成Veo、Jimeng、Midjourney等5+顶尖模型 |
| **组织方式** | Gallery/社区驱动（TapTV），通过"克隆工作流"方式获取他人风格配置 |
| **实现方式** | **模型路由 + 节点工作流 + 导演级镜头控制**。不是预设风格列表，而是通过Cinema Lab画布精确控制 |
| **关键功能** | 三视图生成、九宫格分镜、多角度控制、火柴人动作参考、焦点编辑（魔法棒）、首尾帧过渡、摄影机控制 |
| **角色** | Tapnow Studio是开源节点编辑器（类似ComfyUI），通过可视化节点组合模型能力 |

### 2.3 ArcReel

| 维度 | 详情 |
|------|------|
| **风格系统** | **图像参考驱动**（Style Reference Image）。上传风格参考图→AI自动分析→全局应用 |
| **实现方式** | **Prompt注入 + XML风格块**。后端代码中风格通过 `<style>` XML标签注入prompt |
| **Prompt结构** | `<style>风格：{style}\n描述：{style_description}\n画面比例：{aspect_ratio}</style>` |
| **约束机制** | 硬约束规则："画面元素（材质、装束、道具质感、环境年代特征）须贴合style块定义的风格基调" |
| **反例驱动** | 使用正反例约束（如"赛博朋克风下不出现榻榻米"）而非堆砌规则清单 |
| **开源状态** | MIT License，开源代码可直接参考 |

### 2.4 魔因漫创 (Moyin Creator)

| 维度 | 详情 |
|------|------|
| **六层锚点** | 完全聚焦于**角色一致性**（外观/肤色/服装/表情/行为/声音），**不包含视觉风格层** |
| **画风系统** | **独立于角色系统**，归属分镜模块 |
| **支持风格** | 7种预设：2D动画、3D建模、写实风格、定格动画、水墨风格、赛博朋克、古装仙侠 |
| **实现方式** | **Prompt标签注入**。提示词结构：`[主体] + [场景] + [镜头语言] + [情绪氛围] + [风格标签]` |
| **风格示例** | `cinematic, 4K, wuxia fantasy aesthetic` 或 `traditional Chinese aesthetics` |
| **架构设计** | 角色一致性 vs 视觉风格 → 做了**清晰的工程解耦**。这个设计理念值得参考 |

### 横评对比总结

| 应用 | 风格数量 | 实现方式 | 精度控制 | 能否复用 |
|------|----------|----------|----------|----------|
| 小云雀2.0 | 100+ | 自研模型+Prompt | 高(光影参数面板) | 无法直接复用 |
| TapNow | 基于社区 | 模型路由+工作流 | 最高(逐镜头控制) | 理念可参考 |
| ArcReel | 无限(上传参考) | 图参考+Prompt注入 | 中(依赖参考图质量) | **代码开源,可直接参考** |
| Moyin Creator | 7种预设 | Prompt标签注入 | 低(仅文本描述) | **架构可参考** |

## 四、StarCanvas 100+画风库实现建议

### 4.1 整体策略：分层实现，算力从低到高

```
成本层级          实现方式                      覆盖面    算力消耗
────────────────────────────────────────────────────────
L1 零成本         Prompt文本注入（JSON模板）      100%风格  ~0
L2 低成本         色彩LUT后处理 (.cube)           60%风格   极低
L3 中成本         风格参考图IP-Adapter             40%风格   中等
L4 高成本         模型微调/LoRA                    10%风格   高
```

### 4.2 推荐方案：L1+L2为主，L3为辅

#### L1: Prompt文本注入（覆盖全部100+风格）

**数据来源**：直接从 awesome-seedance (CC BY 4.0) 和 ArcReel 开源代码中提取模板

**实现方式**：
```json
{
  "style_id": "wong-kar-wai-1990s",
  "name": "王家卫90年代香港",
  "category": "真人写实/港风",
  "prompt_injection": {
    "global": "1990s Hong Kong art film aesthetic, Wong Kar-wai style cinematography",
    "color": "yellow-green color palette, high ISO film grain, color shift",
    "motion": "step-printing effect, handheld camera, shallow depth of field",
    "lighting": "neon light, rain-soaked streets, practical lighting",
    "negative": "digital clean look, sharp focus, bright daylight"
  },
  "classify_tags": ["港风", "电影感", "怀旧", "文艺", "雨夜"]
}
```

**优势**：零算力，与任何底层模型兼容，从Seedance到Sora都能用

#### L2: 色彩LUT后处理（约60-80种风格）

**数据来源**：Film-Luts (MIT) + 自制LUT映射表

**高优先级风格**：
- 柯达暖黄胶片 / Fuji冷淡 / Cinestill / 黑白颗粒 / 褪色胶片
- 赛博朋克霓虹 / 蒂芙尼蓝 / 姜黄色调 / 蓝橙对比
- 冬日冷调 / 日落暖金 / 迷雾灰

**实现方式**：
```
1. 将Film-Luts的G'MIC格式LUT转为标准.cube格式
2. 对已生成的视频帧应用3D LUT色彩映射
3. 可使用GPU加速的实时色彩转换（OpenGL shader或Metal）
```

**优势**：算力极低（单帧<1ms at 1080p），无需重复生成视频

#### L3: 风格参考图 + IP-Adapter（高精度控制用）

**使用场景**：当用户对某个特定画风要求极高，需要"像某个电影一样"时

**实现方式**：
```
1. 从FilmNet数据集(MIT)中选取对应风格参考图
2. 使用IP-Adapter进行风格迁移（而非内容迁移）
3. 将IP-Adapter权重作为风格参数的一部分
```

**成本**：约增加到5-10%的推理时间

### 4.3 100+画风的来源分配建议

| 数量 | 来源 | 方式 |
|------|------|------|
| 40+ | awesome-seedance prompt模板 | 直接提取→结构化JSON |

### 4.4 架构设计建议

```
StarCanvas 画风系统架构
├── StyleLibrary (JSON)
│   ├── categories/          # 画风分类索引
│   ├── presets/             # 每个画风的JSON定义
│   │   ├── prompt_block/    # L1: 文本注入模板
│   │   ├── lut_mapping/     # L2: 对应哪些LUT
│   │   ├── reference_imgs/  # L3: 风格参考图URL
│   │   └── constraints/     # 反例约束(参考ArcReel)
│   └── tags/                # 多维度标签体系
│
├── StyleInjector (运行时)
│   ├── PromptInjector       # 将style定义注入生成prompt
│   ├── LUTProcessor         # 后处理色彩LUT映射
│   └── IPAdapterManager     # 可选的高精度风格适配
│
└── StylePipeline (编排)
    └── 根据画风ID自动选择L1/L2/L3策略组合
```

### 4.5 最小可行方案（MVP）

**目标**：2周内实现50+风格，零额外算力

1. **从 awesome-seedance 提取30个电影风格prompt模板**（已有CC BY 4.0授权）
2. **从 Film-Luts 提取15个色彩LUT映射配置**（MIT授权）
3. **手工补充5个中国特色风格**（国风水墨/古装仙侠/90年代农村/旗袍复古/竖屏爽剧）
4. **构建风格JSON库** → 运行时通过Prompt注入 + LUT后处理实现

**这样不需要**：
- 额外模型训练
- LoRA微调  
- GPU服务器
- IP-Adapter引擎

**只需要**：
- 一次性的prompt工程（1-2周）
- 简易的LUT处理管线（已有开源代码）
- 风格JSON的持续维护
