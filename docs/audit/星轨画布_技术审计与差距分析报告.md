## 五、核心竞争力评估

### 5.1 领先优势

1. **分镜导演 AI 系统**：`storyboard-director-agent.ts` (398 行) + `lib/storyboard/` (11 模块) + `cinematic-rules.ts` 形成业界罕见的专业级导演 Agent，包含 15 种情绪→镜头策略、10 种运镜方式、连续性自动检查——这在代码层面远超 TapNow 等竞品的 Prompt 工程深度。

2. **CinematicPromptPipeline**：`packages/canvas/src/prompt-analyzer/` 提供的分析管线（简笔画分析、3D Previs 分析、分镜分析）是独特竞争力。

3. **440 测试全覆盖**：38 个测试文件，涵盖核心引擎、布局算法、分镜合成、图遍历等关键路径。

### 5.2 核心瓶颈

1. **视频管线完全缺失**：从 TapNow 对标来看，这是最大差距。AI 影视创作工具的"最后一公里"是图→视频，目前仅 Mock。

2. **资产系统无持久化**：角色资产库逻辑存在但纯内存，无数据库、无前端 UI，长项目无法维持角色一致性。

3. **无用户系统**：所有项目数据仅存在浏览器，不可分享、不可协作、不可云端恢复。

4. **交互效率差距**：无节点级 `/` 命令、无框选批量操作、无右键菜单——与 TapNow 的交互效率存在代差。

## 七、架构评价与风险提示

### 7.1 架构亮点

1. **Monorepo 结构规范**：Turborepo + pnpm workspace，build/test/typecheck 全部触发式执行
2. **服务端安全**：AI API Key 绝不暴露到浏览器，通过 Next.js API Routes 代理
3. **模块级桥接模式**：跨组件通信方案（模块级变量）虽不优雅但解决了 ReactFlow nodeTypes 注册在组件树外的技术难题
4. **测试驱动**：440 tests 的测试覆盖率在 AI 创作工具中实属罕见
5. **类型安全**：TypeScript strict mode + 跨包类型共享（`@creative-canvas/canvas`, `@creative-canvas/shared`）

### 7.2 架构风险

1. **模块级桥接变量的内存泄漏风险**：`_runAgentFn` / `_runBatchGenerateFn` / `_undoStack` / `_redoStack` 均为模块级可变状态，需确保在组件 unmount 时清理
2. **无服务端数据持久化**：全量数据仅存浏览器，F5 刷新依赖 ReactFlow 内置状态恢复，隐私窗口/清缓存将丢失所有项目
3. **NestJS 后端占位**：`apps/api` 目录存在但从未启用，若计划使用需重写所有 API Routes
4. **AI 模型硬编码**：`gpt-5.5`, `gpt-image-2` 等模型名硬编码在多处，更换 Provider 需改代码

### 7.3 商业模式评估

- `packages/billing/` 目录存在（占位）
- `BillingMode` 枚举（`PLATFORM_CREDITS`, `BYOK_PERSONAL`, `BYOK_ORGANIZATION`）
- **结论**：计费模式已预留三档设计，但未实现，当前为纯 BYOK（Bring Your Own Key）模式
