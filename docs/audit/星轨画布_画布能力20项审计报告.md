## 最紧急的 3 个问题

### 🔴 P0 — 数据持久化不完整
- **问题**：`useCanvasPersistence.ts` 使用 localStorage（4MB 限制），图片被剥离后依赖 IndexedDB 恢复
- **风险**：大画布保存失败；清除浏览器数据后所有工作丢失；无法跨设备同步
- **建议**：实现 Prisma + PostgreSQL 服务端持久化；或增加 IndexedDB 全画布快照方案

### 🔴 P0 — 无服务端数据层
- **问题**：`apps/api/src/prisma/` 中无 `schema.prisma` 文件，所有数据仅存在浏览器端
- **风险**：无法多用户协作；无法做版本历史；无法做自动保存冲突解决
- **建议**：创建 `prisma/schema.prisma`，定义 User、Project、Canvas、Node、Edge、Asset 模型

### 🟠 P1 — 节点删除/复制 UI 缺失
- **问题**：`canvasStore.ts` 中有 `clipboardNode` 框架，但无 UI 入口；删除仅能通过 React Flow 快捷键
- **风险**：普通用户不知道如何删除/复制节点
- **建议**：实现右键菜单（`NodeContextMenu.tsx`）；增加 Ctrl+C/Ctrl+V 快捷键处理

## 技术亮点（值得保持）

1. **IndexedDB 图片存储方案**（`localImageStore.ts`）
   - 图片 Blob 不进 localStorage，避免 4MB 限制
   - 迁移逻辑：自动将旧 base64 数据转入 IndexedDB

2. **分镜导演 Prompt 工程**（`storyboard-director-agent.ts`）
   - 4步工作流 + 15种情绪→镜头策略
   - 结构化输出解析 + 降级处理

3. **SanitizePersistedCanvas**（`sanitizeNodesForPersistence.ts`）
   - 防止运行时 URL（blob:/data:）污染持久化数据
   - 开发模式泄漏检测
