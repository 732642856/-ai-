- 将每个 ShotNode 整理为统一生产简报，包含：
  ```
  visual：画面提示词 / 景别 / 运镜 / 时长 / 角色一致性资产
  voice：对白 / 配音意图 / 声音提示 / TTS 建议文本
  subtitle：字幕文本 / 字幕意图
  handoff：后期交接说明 / 连续性与风险警告
  ```
- **StarCanvas 接入：** 项目包导出新增 `productionBriefs`

### 7. Project Package v1.1 manifest（生产运行清单）
- **文件：** `src/lib/storyboard/projectPackageManifest.ts` + `.test.ts`
- 借鉴 ComfyUI_frontend 的 queue-by-shot / progress manifest 思路
- 生成 `productionRunManifest`：
  ```
  workflow stages（剧本 → 分镜 → 视觉 → 声音 → 字幕 → 合成 → 后期交接）
  counts（总 shot 数、有 prompt 数、有配音数、有字幕数）
  shotBriefIndex（每个 shot 的 id/title/actions 快速索引）
  productionRunPlan（每个 shot 的下一步动作列表）
  handoffWarnings（所有需要人工处理的警告）
  assetLinks（视觉参考、声音意图、交接说明节点的引用 ID）
  ```
- **StarCanvas 接入：** 项目包导出新增 `productionRunManifest`

### 8. Production Run Queue v1（生产队列纯状态模型）
- **文件：** `src/lib/storyboard/productionRunQueue.ts` + `.test.ts`
- 从 `productionRunManifest.productionRunPlan` 生成可执行队列
- **支持动作：**
  - 可自动执行：`generate-storyboard-image`、`generate-voice-track`、`create-subtitle-track`、`review-handoff-warnings`
  - 需手动解锁：`add-visual-prompt`（缺少视觉 prompt 的镜头）
- **状态机：** `queued → preparing → running → completed / failed`
- **统计：** `totalTasks`、`completedTasks`、`failedTasks`、`progress`、`activeTaskId`

### 关于 OmniVoice TTS（重要背景）

一楠此前深入调研过 [OmniVoice (k2-fsa)](https://github.com/k2-fsa/OmniVoice) 开源语音合成项目。
OmniVoice 是一个 zero-shot TTS 模型，支持 **voice cloning**（克隆语音）和 **voice design**（语音设计）两种模式，
目标是集成到星轨画布的分镜节点中，为每个角色赋予独立声线。

当前状态：
- `ttsService.ts` 已有基础 TTS 接口，`VoicePanel.tsx` 已有配音面板 UI
- `CinematicShot` 已有 `dialogue`、`voiceIntent`、`soundCue` 字段
- 上述三者在 StarCanvas 中已具备结构支撑，但尚未与 OmniVoice 实际对接
- 续接时如需推进配音链路，建议先复用现有 `ttsService.ts` 接口层，再逐步接入 OmniVoice
