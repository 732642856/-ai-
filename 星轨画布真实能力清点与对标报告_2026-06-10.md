## 一、TapNow真实流程复盘（用户视角的16步操作）

基于对TapNow截图和文档的深度分析，一个真实用户从打开到出片的完整流程是：

```
打开黑色网格画布
  ↓
左侧工具栏上传角色图/产品图 → 画布上出现图片节点
  ↓
右侧Greeting面板@引用图片 → AI读取并分析素材
  ↓
新建Text节点写下视频目标（30秒竖屏科幻短片）
  ↓
底部Gemini输入框生成脚本 → 产出ScriptNode
  ↓
AI拆分为6个分镜节点（StoryboardNode）
  ↓
每个分镜连接角色参考图 → 创建生图节点 → 生成分镜图
  ↓
分镜图连接视频节点 → 生成6个视频片段
  ↓
添加旁白Text节点 → 连接语音节点生成音频
  ↓
添加字幕节点 + 背景音乐节点
  ↓
全部接入合成节点 → 生成最终视频
  ↓
哪里不满意，回到对应节点局部重跑
```

**TapNow的核心设计哲学**：
1. **节点即流程** —— 不是装饰，每个节点代表一个真实生产步骤
2. **Chat即指挥** —— AI读取画布、理解节点关系、生成新节点、驱动执行
3. **局部重跑** —— 修改一个分镜，只需重跑该分镜对应的生图+视频节点
4. **专业控制** —— Cinema Lab提供多角度拖拽调姿态、火柴人动作参考、魔法棒焦点编辑
5. **社区即教程** —— TapTV社区，12万+可克隆工作流

### 3.2 核心差距（P0级）

1. **Chat不能操作画布** —— 这是TapNow的核心体验。用户说"帮我生成分镜"，TapNow真的创建节点。星轨画布的Chat只能"看到"不能"动手"。
2. **缺少视频合成引擎** —— 星轨能生成多段视频片段，但没有把它们+音频+字幕合成为一个MP4的能力。
3. **没有社区/模板生态** —— TapNow的TapTV有12万+可克隆工作流，这是巨大的用户粘性来源。
4. **角色三视图前端UI** —— API已就绪，但缺少独立参数面板。

### 3.3 星轨独有优势

1. **ContinuityGuard六维连续性** —— TapNow完全没有。这是影视制作的核心质量保障。
2. **角色Bible六层身份锚点** —— 从骨骼结构到signature scent的深度角色管理。
3. **Vidu 8种模型支持** —— 比TapNow的视频模型选择更丰富。
4. **开源+本地化** —— 全代码开源，无平台锁定。

## 五、我们应该如何向用户展示真实能力？

### 5.1 当前可以真实演示的完整流程（MVP链路）

**建议演示路径**：
```
1. 打开星轨画布 → 黑色网格画布
2. 上传角色三视图 → ImageNode出现
3. 右侧Chat输入："分析这张角色图"
4. 新建Text节点 → 写下视频目标
5. 底部AI输入框："基于这个角色，生成30秒科幻短片脚本"
6. AI生成ScriptNode
7. 运行工作流 → 自动拆分6个分镜节点
8. 每个分镜连接角色图 → 生图节点 → 生成分镜图
9. 分镜图连接视频节点 → 调用Vidu生成视频片段
10. 添加旁白Text节点 → 连接语音节点生成音频
11. 展示最终多段视频+音频结果
```

**注意事项**：
- 视频合成（多段合为一个MP4）需要说明"正在开发中"
- Chat不能自动创建节点，需要手动操作
- 角色三视图需要在CinemaLabPanel中手动切换角度

### 5.2 建议的展示策略
